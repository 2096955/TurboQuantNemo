# Phase B Implementation Plan

> **ARCHIVED:** Superseded by IsoQuant decode-perf plan (2026-04-24). Phase B goals
> (batching, V compression, Rust backend) were replaced by the IsoQuant quaternion-rotation
> approach with Metal kernels.

**For agentic workers:** REQUIRED: Use superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver memory savings and throughput for TurboQuant on MLX by batching per-head operations, compressing values, and writing a Rust storage backend.

**Architecture:** We will refactor `TurboQuantKVCache` to store compressed keys and values in batched tensors rather than per-head dictionaries, enabling a single `asymmetric_attention_scores` broadcast. We will implement MSE-only scalar quantization for values to match keys' footprint without QJL overhead. Finally, we will introduce a Rust PyO3 extension (`turboquant_storage`) to bit-pack the 3-bit indices and sign bits, bypassing Python object overhead.

**Tech Stack:** Python, MLX, Rust (PyO3)

```python
    def compress_value(self, x: mx.array, rotation: mx.array) -> dict:
        x_f32 = x.astype(mx.float32)
        x_norm = mx.linalg.norm(x_f32, axis=-1, keepdims=True)
        x_unit = x_f32 / mx.maximum(x_norm, mx.array(1e-8))
        Q_T = mx.transpose(rotation)
        x_rot = mx.matmul(x_unit, Q_T)
        indices, _ = quantize_scalar(x_rot, self.centroids, self.boundaries)
        return {"indices": indices, "x_norm": x_norm}

    def decompress_value(self, compressed: dict, rotation: mx.array) -> mx.array:
        x_rot_quant = self.centroids[compressed["indices"]]
        x_hat_unit = mx.matmul(x_rot_quant, rotation)
        return x_hat_unit * compressed["x_norm"]
```
